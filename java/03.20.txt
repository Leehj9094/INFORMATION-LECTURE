람다식(Lamda)?
- 이름 없는 함수(익명함수)
js -> ( )=>{ }
java -> ( )->{ }.stream( ).forEach( )
- 람다식 자체만으로도 메서드 역할을 대신할 수 있으며, 메서드의 매개 변수로 전달
되어지는 것도 가능, 메서드가 결과로 반환될 수도 있음

스트림(Stream)?
- 원본 데이터를 변경하지 않고 읽기만 함
- 작업을 내부 반복처리 함
- 병렬 처리가 쉬움(multi Thread 사용)
- 기본형 스트림을 제공(Stream<Integer>)

함수인터페이스(FunctionalInterface)?
- 1개의 추상 메소드를 갖는 인터페이스
- @FunctionalInterface 를 사용 => 컴파일러에게 전달 해서 인터페이스가 함수형 인터페이스 조건에 맞는지 검사해줌
- 람다식을 제공하기 때문에 요약? 간단하게 쓸수 있음
